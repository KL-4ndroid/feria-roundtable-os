# Compaction Reports｜文件減肥報告區

Store completed compaction reports here.

Reports should explain what was shortened, what was preserved, what was moved to archive, and how meaning was protected.

---

## Naming convention

```text
YYYY-MM-DD_PROJECT-ID_short-description.md
```

Example:

```text
2026-07-01_P-001_initial-compaction.md
```

---

## Report should include

- project name
- files reviewed
- reason for compaction
- core content preserved
- disagreements and risks preserved
- content moved to archive
- updated compacted summary
- meaning preservation checklist

Use:

```text
03_templates/slimming_template.md
```
