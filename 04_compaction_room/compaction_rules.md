# Compaction Rules｜文件減肥規則

The Compaction Room exists to keep long-running project files usable.

It reduces document size without changing meaning.

---

## Core principle

> Reduce words, not meaning. Remove noise, not evidence.

---

## What the Compaction Room should do

1. Remove repeated explanations
2. Remove low-value chatter
3. Merge duplicate conclusions
4. Separate confirmed decisions from assumptions
5. Preserve objections and risks
6. Preserve important original wording
7. Move raw material to archive
8. Update compacted summaries
9. Record what changed
10. Make the next role's job easier

---

## What the Compaction Room must not do

The Compaction Room must not:

- make new strategic decisions
- delete objections because they are inconvenient
- remove important risks
- change a role's original meaning
- turn assumptions into facts
- turn recommendations into decisions
- delete raw material permanently
- make the project look more certain than it is

---

## Compaction triggers

Start compaction when:

- a file has more than 500 lines
- a file has more than 30 KB
- the same point appears more than three times
- roles need to reread too much history
- a decision is buried in conversation
- a project changes phase
- the project is being handed to another role
- the Control Room asks for a current summary

---

## Required preservation

Always preserve:

- final decisions
- reasons behind decisions
- rejected options
- paused directions
- objections
- risks
- unresolved questions
- original user constraints
- important quotes or user wording
- next actions

---

## Output files

The Compaction Room usually updates:

- project `07_compacted_summary.md`
- project `08_changelog.md`
- project `99_raw_archive.md`
- optionally project `00_project_brief.md`

---

## Compaction quality checklist

Before finishing, check:

- [ ] Did I preserve the original intent?
- [ ] Did I keep the current decision status clear?
- [ ] Did I preserve objections and risks?
- [ ] Did I distinguish facts, assumptions, recommendations, and decisions?
- [ ] Did I move raw details to archive instead of deleting them?
- [ ] Did I make the next role's task clearer?

---

## Warning

A short but distorted summary is worse than a long messy log.

When unsure whether something is important, preserve it and mark it as uncertain.
