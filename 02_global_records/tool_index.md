## Free tool criteria

A free tool should:

1. solve one small, clear problem
2. give results quickly
3. require minimal user input
4. produce immediately usable output
5. connect naturally to a template, service, or product hypothesis
6. avoid becoming unlimited free custom work

## Maintenance rule

Update this index when:

- a new free tool project is created
- MVP scope changes
- a tool is launched
- a tool is paused
- a tool becomes evidence for a product feature
