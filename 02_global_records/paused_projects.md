# Paused Projects｜暫緩專案清單

Use this file to record intentionally paused projects.

A paused project is not a failure. It is a project that should not consume attention right now.

---

## Format

| Project ID | Project name | Type | Pause date | Pause reason | Resume condition | Review timing | Notes |
|---|---|---|---|---|---|---|---|
|  |  |  | YYYY-MM-DD |  |  |  |  |

---

## Pause reasons

Common reasons:

- Not enough validation
- Too broad
- Too expensive to build
- Too early for current stage
- Better as content first
- Better as a template first
- Better as a manual service first
- Merged into another project
- Not aligned with current strategy

---

## Resume rule

A paused project can resume only when at least one resume condition is met, such as:

- repeated user requests
- strong content response
- free tool usage evidence
- template sales evidence
- consulting or service feedback
- strategic priority change

The Control Room should approve any resumed project.
