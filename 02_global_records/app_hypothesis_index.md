Use this file to track product or app feature hypotheses.

This is not a development backlog. It is a validation pool.

---

## Product hypothesis criteria

A feature hypothesis should answer:

1. What real pain point does this come from?
2. Has this need appeared repeatedly?
3. Has it been validated through content, tools, templates, or service work?
4. Would users use this repeatedly?
5. Is it better than a spreadsheet, document, or conversation?
6. Can a small team build the minimum version?
7. Is there a simpler non-product solution?

---

## Maintenance rule

Update this index when:

- a new product feature hypothesis is created
- validation evidence changes
- a hypothesis enters or leaves MVP consideration
- a feature is paused, merged, or retired
